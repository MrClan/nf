NHRC Web Frontend Revamp

Why revamp? (12 major reasons)

  1. Modernization (with latest technologies and trends)
  2. Current version is outdated and cluttered with issues (mentioned in later slides)
  3. Make usable on all devices - desktop and phones
  4. Improved user experience
  5. Support multi-language content(extendable to any number of languages)
  6. Easier and streamlined content management
  7. RBAC: Multi-level editors - for respective content type(newseditor, noticeeditor, admin)
  8. Content Review Flow
  9. Automatic content archival
  10. End user bandwidth savings
  11. Better performance
  12. Robust Security

Built for Everyone, Everywhere

Modern in spirit, classic in foundation

  1. ✅ Puts content first – Lightweight, fast-loading pages that focus on information, not flashy effects
  2. ✅ Designed for everyone – Clear typography, high contrast, and intuitive layouts for all users
  3. ✅ Respects all users – Works on any device, from smartphones to legacy phones like Nokia S40
  4. ✅ Prioritizes accessibility – Built with semantic HTML, keyboard navigation, and screen reader support
  5. ✅ Embraces simplicity – No JavaScript bloat, no broken experiences when features fail
  6. ✅ Enhances progressively – Modern features (animations, dark mode) are added only if supported
  7. ✅ Ensures reliability – Fast, secure, and always available — critical for public services
  8. ✅ Balances innovation & inclusion – Feels contemporary, but never excludes those with older tech or disabilities

Issues in current version

Addressing Critical UX, Performance & Trust Issues

  1. Unusable on mobile phones - UI breaks, several menus inaccssible on small screens
  2. Outdated & inaccessible features — not screen-reader friendly, hard to read
  3. Unoptimized media - Serves large, high-resolution images to all devices — slows load times, wastes data, especially on mobile
  4. Poor visual hierarchy - Critical notices hidden in collapsible accordions — reduces visibility and delays access to urgent information
  5. Non-responsive image handling - Images stretched or pixelated — undermines credibility and appears unprofessional on mobile
  6. Hard to maintain, extend, or audit — increases long-term risk and cost
  7. Uncertain backend security & maintenance - WordPress setup with unknown plugin origins, no clear update policy, and potential vulnerabilities
  8. Slow performance

 

This isn’t just a redesign — it’s a necessary upgrade to meet public expectations for speed, accessibility, and trust.
